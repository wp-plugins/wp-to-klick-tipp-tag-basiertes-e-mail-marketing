Bestell-Status</li><li>Keine Synchronisation wenn der Bestell-Status sich ändert z.B. abgebrochen, zurückerstattet etc.</li><li>Keine Synchronisation der gekauften Produkte/Kategorien etc.</li></ul><blockquote>Premium Plugin Version:Synchronisiert und erstellt folgende Tags:<ul><li>Synchronisiert aller vorhandenen Wordpress Benutzerrollen</li><li>Bestellstatus</li><li>Produkte die gekauft wurden (bei Bestellungen mit dem Status "completed"/abgeschlossen)</li><li>Produktkategorien</li><li>Umsatz je Bestellung in 50 € Schritten</li><li>Eingekauft in den letzten 6 Monaten</li></ul>+ Synchronisierte Daten über die Extra Felder: Kundenname, Firma, Adresse, Telefon<br>+ Synchronisierte Daten über die Extra Felder (der letzten Bestellung): Bestellnummer, Bestellstatus, exakter Umsatz, Produkte, Versandkosten, Rabatte<strong>Tipp:</strong> Wenn Sie in der Klick-Tipp Kontakt Cloud nach Extra Felder filtern, können Sie dem Ergebnis manuell tags zuweisen<br><br></blockquote>Synchronisierungsablauf am Beispiel: Kauft der Kunde in Ihrem Shop wird er automatisch zu Klick-Tipp synchronisiert inkl. double opt-in.Je nach Zahlungsart bekommt er den Bestellstatus, "Wartend", "In Bearbeitung", ...<br>Weitere Tags werden dem Benutzer hinzugefügt wenn sich der Bestellstatus auf "Abgeschlossen" ändertz.B. <ul><li>Produkte die gekauft wurden</li><li>Produktkategorien, </li><li>Umsatz je Bestellung in 50 € Schritten</li><li>Eingekauft in den letzten 6 Monaten</li></ul>Wird eine Bestellung gutgeschrieben/abgebrochen wird auch der Bestellstatus sich ändern und Tags ggf. entzogen.Technik: WP-Cron wird zum auslösen der Synchronisierung verwendet.Die Synchronisierung erfolgt über die REST API Schnittstelle. REST API wird sowohl von WooCommerce als auch von Klick-Tipp von Haus aus unterstützen.<br><blockquote><h4>Unterstützung:</h4> Dieses Plugin wurde von einem der ersten
